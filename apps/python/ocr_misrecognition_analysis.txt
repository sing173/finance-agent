OCR 文本反向识别问题分析
================================================================================

问题：'户名'和'账号'被识别为'名户'和'号账'

根本原因：OCR 引擎将单个汉字分割成独立的文本块
  - '户名' → ['户', '名'] 两个独立块
  - '账号' → ['号', '账'] 两个独立块

问题发生在文本排序阶段：
  当前排序: blocks.sort(key=lambda b: (b['y0'], b['x0']))
  排序依据: 左上角坐标 (y0, x0)

受影响的行（rows 1-2，col 1 和 col 5）:
--------------------------------------------------------------------------------

Row 1, Col 1 (户名):
  按 (y0, x0) 排序: ['户', '名']
  按 (cy, cx) 排序: ['户', '名']
    '户' at (451,359)-(499,405), center=(475,382)
    '名' at (523,359)-(571,408), center=(547,384)

Row 2, Col 1 (账号):
  按 (y0, x0) 排序: ['号', '账']
  按 (cy, cx) 排序: ['号', '账']
    '号' at (527,472)-(571,522), center=(549,497)
    '账' at (451,474)-(499,520), center=(475,497)

Row 2, Col 5 (账号):
  按 (y0, x0) 排序: ['号', '账']
  按 (cy, cx) 排序: ['账', '号']
    '号' at (1461,472)-(1507,522), center=(1484,497)
    '账' at (1387,474)-(1436,518), center=(1411,496)

详细分析：
--------------------------------------------------------------------------------

1. '户名' 问题（Row 1, Col 1）:
   - '户': y0=359, cx=475
   - '名': y0=359, cx=547
   - y0 相同，按 x0 排序: 475 < 547 → '户'在前 → 合并为'户名' ✓ 正确

2. '账号' 问题（Row 2, Col 1）:
   - '号': y0=472, cx=549
   - '账': y0=474, cx=475
   - y0 不同: 472 < 474 → '号' 在前 → 合并为'号账' ✗ 错误！
   - 原因: OCR 对 '账' 的检测框 y0 偏大 2 像素，导致排序错误

3. '账号' 问题（Row 2, Col 5）:
   - '号': y0=472, cx=1484
   - '账': y0=474, cx=1411
   - y0 不同: 472 < 474 → '号' 在前 → 合并为'号账' ✗ 错误！
   - 原因: 同上

解决方案：
--------------------------------------------------------------------------------

方案1: 改用 (cy, cx) 排序 - 使用中心点代替左上角
  - '号': cy=497, cx=549
  - '账': cy=497, cx=475
  - cy 相同，按 cx 排序: 475 < 549 → '账'在前 → '账号' ✓

方案2: 合并时检测反向并纠正
  - 如果 merged text 在已知的误识别列表中，交换字符顺序
  - 如 '号账' → '账号', '名户' → '户名'

方案3: 使用 y-center 进行行分组，行内按 x-center 排序
  - 更稳定的坐标点，对微小 y 偏移不敏感
  - 同时可以容忍不同字符检测高度的差异
