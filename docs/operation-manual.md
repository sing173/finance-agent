# FinanceAssistant 操作手册

## 一、快速开始

**一句话流程：** 拖文件 → 自动识别 → 解析 → 凭证预览 → 导出

```
① 启动应用 → ② 拖入银行流水文件 → ③ 点击「开始解析」
                                         ↓
④ 点击「凭证生成」→ ⑤ 检查/修正科目 → ⑥ 确认导出
```

### 首次使用前

1. 导入科目表：点击「导入科目」→ 选择金蝶导出的 xlsx 科目表（约 300 条标准科目）
2. 配置账号映射：点击「账号管理」→ 添加银行账号对应的会计科目

## 二、文件解析

### 2.1 选择文件

| 方式 | 操作 |
|------|------|
| **拖拽** | 把文件拖到中间拖拽区域 |
| **浏览** | 点击拖拽区域，从文件夹选择 |
| **批量** | 一次拖入多个文件（最多 5 个），自动去重 |

**支持格式：**

| 银行 | 流水 PDF | 回单 PDF | CSV | Excel |
|------|----------|----------|-----|-------|
| 工商银行 | ✅ | ✅ | ✅ | — |
| 招商银行 | ✅ | ✅ | — | ✅ |
| 广发银行 | ✅ | — | — | — |

### 2.2 解析流程

```
拖入文件 → 自动检测银行 → 解析交易明细 → 展示结果
               │                │
         失败→手动选择      失败→检查文件格式
```

- **银行检测**：系统自动识别银行和文档类型，失败时弹出手动选择窗口
- **交易明细**：解析完成后展示列表，支持排序/筛选/分页，顶部显示收支统计
- **OCR 扫描件**：回单/扫描件自动触发 OCR（约 1-2 秒/页），耗时较长

### 2.3 批量处理

```
添加多个文件 → 识别文件（逐个检测银行类型）
                → 开始解析（逐个解析，显示进度）
                → 查看结果摘要（成功/失败/总笔数）
```

- 检测失败的文件可点击「修改配置」手动指定银行
- 解析失败的文件显示红色，可修改后重试
- 成功文件可合并导出统一凭证

## 三、凭证生成（核心功能）

### 3.1 进入凭证预览

解析完成后点击「凭证生成」→ 进入全屏预览面板。

### 3.2 预览面板说明

每条凭证包含多条分录，示例：

```
凭证 #1  日期: 2026-03-31  方向: 支出
  分录1  借  管理费用_物业管理费    ¥1,200.00  部门: 公共部门
  分录2  贷  银行存款_工行基本户    ¥2,400.00
         ↑ 匹配来源: rule（规则匹配）
```

**界面元素：**

| 元素 | 说明 |
|------|------|
| 凭证编号 | 自动编号，同类交易合并为一张凭证 |
| 科目名称 | 黑色=自动匹配，红色⚠=未匹配 |
| 匹配来源 | `rule`=规则匹配 / `history`=历史学习 / `manual`=手动修正 |
| 部门列 | 管理费用自动填充"公共部门"（辅助核算） |

### 3.3 科目匹配机制

系统使用**三层匹配**，自动程度逐级降低：

```
L1 规则匹配 → L2 历史学习 → L3 手动选择
  │               │              │
 命中即停      命中即停       最后兜底
```

| 层级 | 匹配方式 | 准确度 |
|------|----------|--------|
| **L1 规则** | 根据摘要关键词匹配预设规则（如"物业费"→物业管理费） | 高 |
| **L2 历史** | TF-IDF 相似度匹配历史手动修正记录（阈值 0.75） | 中 |
| **L3 手动** | 用户手动选择科目（红色高亮条目） | — |

> **注意**：只有手动修正的结果会进入历史库供 L2 学习，自动匹配结果不记录。用得越多，L2 匹配越准。

### 3.4 手动选择科目

1. 点击红色高亮条目 → 弹出科目选择器
2. 搜索关键词或按类别筛选 → 点击选择
3. 选择后自动更新，匹配来源变为 `manual`

### 3.5 批量填充

上方下拉菜单提供常用预设科目，选择后一键应用到所有未匹配条目。

### 3.6 辅助核算（部门列）

- 管理费用（50602xx 系列科目）自动填充部门为"公共部门"
- 导出 Excel 时写入第 15 列（部门列），金蝶导入后自动识别
- 银行分录和收入类科目不填充部门

### 3.7 保存/加载草稿

| 操作 | 说明 |
|------|------|
| **保存草稿** | 未完成可暂存，下次继续编辑 |
| **加载草稿** | 从历史草稿列表选择恢复 |
| **草稿持久化** | 跨会话不丢失，存储在本地 SQLite 数据库 |

### 3.8 确认导出

1. 检查所有条目均有科目（无红色⚠标记）
2. 点击「确认导出」→ 选择保存位置
3. 导出文件为金蝶精斗云 25 列凭证导入模板

## 四、账号与科目管理

### 4.1 银行账号→科目映射

**入口：** 主界面「账号管理」按钮

| 操作 | 说明 |
|------|------|
| **新增** | 选择银行代码 → 匹配方式（后缀/精确）→ 输入账号模式 → 选科目 |
| **修改** | 表格中直接编辑 |
| **删除** | 表格中删除 |

**匹配规则：**

- **精确匹配**：账号完全一致才匹配
- **后缀匹配**：账号末尾匹配指定模式（如尾号 4363 → 工行基本户）

### 4.2 导入科目表

首次使用需导入金蝶导出的科目表（xlsx 格式），系统内置约 300 条标准科目。

## 五、数据导出

| 导出类型 | 作用 | 操作 |
|---------|------|------|
| 流水明细 Excel | 原始交易列表 + 统计 | 解析完成后点击「导出明细」 |
| 金蝶凭证 Excel | 25 列凭证导入模板 | 凭证预览 → 确认导出 |

### 凭证 Excel 列说明

| 列号 | 名称 | 说明 |
|------|------|------|
| 1-10 | 日期/凭证字/凭证号/摘要/科目/金额 | 基础凭证信息 |
| 15 | 部门 | 管理费用自动填充辅助核算 |
| 23 | 原币金额 | 交易原始金额 |
| 24-25 | 币别/汇率 | 默认 RMB/1.0 |

## 六、常见问题（FAQ）

### Q1: "无法识别银行类型"怎么办？

- PDF 文件可能不是支持银行的格式
- 扫描件需 OCR 识别（约 1-2 秒/页）
- 手动选择银行 + 文档类型即可强制解析

### Q2: 科目匹配不准确怎么办？

- 在凭证预览中手动修正 → 系统自动学习
- 用得越多，L2 历史匹配越准
- 可在「账号管理」中配置银行账号映射

### Q3: 导出的 Excel 怎么导入金蝶？

金蝶精斗云 → 凭证管理 → 导入 → 选择导出的文件，自动匹配日期、凭证字、科目、金额、部门。

### Q4: 数据存在哪里？

- 所有数据存储在本地电脑，不上传云端
- 数据库位置：`%APPDATA%/FinanceAssistant/data.db`
- 草稿、历史记录、账号映射均在本地

### Q5: 辅助核算（部门）如何生效？

- 系统自动为管理费用科目填充"公共部门"
- 如需修改部门，可在 `subjects.json` 中调整 `aux_category` 配置
- 金蝶导入时自动识别部门列

### Q6: 支持哪些文件格式？

| 格式 | 说明 |
|------|------|
| PDF | 银行流水/回单（文本型 + 扫描件 OCR） |
| CSV | 工商银行对账流水（GBK 编码） |
| Excel | 招商银行流水 |
